# Smart Sprinkler Control — Developer Setup Guide

Everything you need to run, edit, test, and deploy this integration from a cold start.

---

## Prerequisites

- **Docker** via [Colima](https://github.com/abiosoft/colima) on Mac. Start it before anything else:
  ```bash
  colima start
  ```

- **Gotcha — mock `docker` binary**: A fake docker script may be shadowing the real one at `/private/tmp/fake-docker-bin/docker`. The real Homebrew binary lives at:
  ```bash
  ls /opt/homebrew/Cellar/docker/*/bin/docker | head -1
  ```
  If `docker --version` looks wrong or commands fail, prefix with the full path above (or fix your `$PATH` to exclude `/private/tmp/fake-docker-bin`).

## Bring Up the Dev Instance

From the repo root:

```bash
docker compose -f docker-compose.dev.yml up -d
```

- Serves at **http://localhost:8123**. First boot takes 1–3 min.
- Container name: `ha-sprinkler-dev`.

**What's mounted live:**

| Host path | Container path | Purpose |
|-----------|---------------|---------|
| `./custom_components` | `/config/custom_components` | Integration source — edit in place |
| `./dev-config` | `/config` | HA configuration, secrets, etc. |

`dev-config/configuration.yaml` auto-loads the integration with 6 zones and `trusted_networks` auth — **no login or token needed** for the frontend.

**Panel URL:** http://localhost:8123/smart-sprinkler-control

## Editing and Reloading

### Python (integration source)

Files under `custom_components/` are mounted live, but HA caches imports at startup. After any Python change:

```bash
docker compose -f docker-compose.dev.yml restart
```

### Panel JavaScript

**Two copies exist and both must stay in sync:**

| Copy | Path |
|------|------|
| Source of truth (edit here) | `smart-sprinkler-control-panel.js` (repo root) |
| Served by the integration | `custom_components/smart_sprinkler_control/frontend/dist/smart-sprinkler-control-panel.js` |

The integration serves JS from `frontend/dist/` (see `api/http.py` → `frontend_path`). After editing the root copy, sync it:

```bash
/bin/cp -f smart-sprinkler-control-panel.js custom_components/smart_sprinkler_control/frontend/dist/smart-sprinkler-control-panel.js
```

> **Use `/bin/cp -f`**, not `cp` — `cp` is aliased to interactive mode on most Mac setups.

Panel JS changes take effect on a **browser refresh only** — no container restart needed. The panel URL includes a `?v=<content-hash>` cache-buster (generated by `frontend/panel.py`), so browsers auto-refetch on change.

> **Known cleanup:** the root↔dist duplication should be deduped into a single canonical location. Sync manually until then.

## Running Tests

Tests live in `tests/`. They require `pytest`, `pytest-cov`, and HA test deps. Use a gitignored local venv:

```bash
python3 -m venv venv
venv/bin/pip install pytest pytest-cov
# If tests import homeassistant internals, also:
# venv/bin/pip install homeassistant
```

Run:

```bash
venv/bin/pytest
```

**`pytest.ini` config (verified):**

- `--cov` targets `custom_components.smart_sprinkler_control`
- `--cov-fail-under=0` — threshold is currently 0; raise it as coverage grows.
- HTML report written to `htmlcov/`.

## Testing the Precipitation / Rain Feature Locally

The `/api/smart_sprinkler_control/precipitation` endpoint integrates sensor history into 24 hourly buckets using HA's configured local timezone (`America/New_York` in dev).

**Sensor discovery:** the integration matches sensors by `device_class` in `("precipitation", "precipitation_intensity")`, with an OpenWeatherMap entity-name fallback.

### Gotcha: constant sensor values integrate to zero

The endpoint queries the recorder's history and computes the area under the curve over intervals. If a sensor reports a **steady constant value**, the recorder logs only a single row — there is no interval to integrate, and the result is **0**.

**To test properly:** inject a sensor with a **varying** value. A `trigger:` template that nudges the value every ~20 seconds works well:

```yaml
# dev-config/configuration.yaml example
template:
  - trigger:
      - platform: time_pattern
        seconds: "/20"
    sensor:
      - name: "Test Rain Sensor"
        device_class: precipitation_intensity
        unit_of_measurement: "mm/h"
        state: "{{ (range(1, 10) | random) / 10.0 }}"
```

After injecting the sensor, **wait 3–4 minutes** for the recorder to accumulate multiple rows, then hit the endpoint:

```bash
curl http://localhost:8123/api/smart_sprinkler_control/precipitation
```

## Deploy to Production

Production HA runs on a Pi at `10.0.1.11`. The integration is served from a git clone symlinked into HA's config:

- Clone location on Pi: `/config/plugin-sources/ha-smart-sprinkler-control`
- Symlinked to: `/config/custom_components/smart_sprinkler_control`
- Remote: Gogs `http://10.0.10.70/jsugamele/ha-smart-sprinkler-control.git` (branch: `master`)

**Deploy flow:**

```bash
# 1. Commit and push to Gogs master (from your local machine)
git push origin master

# 2. SSH to the Pi and pull
ssh 10.0.1.11
git -C /config/plugin-sources/ha-smart-sprinkler-control pull

# 3. Apply changes:
#    Python changes → restart HA core (startup-safe):
ha core restart

#    Panel-only JS changes → pull is sufficient; cache-buster handles the browser.
```
